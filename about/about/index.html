    📖 关于本站：做一份“活”的 Java Web 指南¶

    写在前面的话

    大家好，我是 Chende。欢迎来到 javaweb.chende.top。

    每开启新一轮的《Java Web 开发技术》教学时，我都会面临一个灵魂拷问：“怎样的Java Web开发教材才是好的教材？”

    ❓ 为什么要做这个网站？¶

    在过去的教学和带班经验中，我发现很多同学在初学 Java Web 时，都会经历一段极其痛苦的“阵痛期”。

    大家在之前的课程中习惯了 System.out.println() 这种在黑框框里所见即所得的单机编程。但一进入 Web 开发的世界，突然就要面对 Tomcat 服务器、HTTP 协议、Servlet 映射、JSP 渲染以及数据库交互。知识点变得极其立体和错综复杂。

    而传统的纸质教材往往存在几个致命痛点：
    1. 技术滞后：书本更新速度太慢，还在教过时的配置方式，而真实企业早就用上了 JDK 17+、Maven 构建和自动化部署。
    2. 过于刻板：满篇的 API 定义和枯燥的理论，很少告诉你“这东西在真实业务里到底是怎么用的”。
    3. 断层严重：默认大家的 Java 基础完美无瑕，但实际上很多同学对“面向对象、集合、反射”等底层内功掌握得并不扎实，导致看不懂框架源码。
    4. 脱离 AI 时代浪潮：当今大模型技术日新月异，各类优秀的 AI 编程助手正在重塑软件开发模式。如果还是闭门造车、死磕传统代码，效率极低。传统的教材对此只字不提，但我希望带大家打破传统的代码壁垒，真正拥抱 AI 热潮，将大模型作为开发中的最强辅助。

    因此，我决定抛弃传统的 PPT 和死板的文档，亲手打造这个开源、在线、持续更新的电子教程网站。

    🎯 这份文档有什么不同？¶

    我希望这份电子文档不仅仅是一本教材，更是你探索 Java 与智能开发宇宙的“导航仪”：

    • 🪵 夯实基础的“补给站”：文档中特意开辟了【Java 基础补给站】专栏。从集合的运用到反射的黑魔法，从文件 I/O 到 JUnit 测试，用最通俗的生活比喻，帮你把千疮百孔的底层内功重新补齐。
    • ⚔️ 实战驱动的“大练兵”：告别“Hello World”式的纸上谈兵。所有的核心知识点（如 Servlet、JSP、MVC 架构）最终都会汇聚到真实的综合实战案例中（如 SmartBook 图书管理模块的 CRUD），让你写出的每一行代码都有真实的业务意义。
    • 🚀 面向未来的“跳板”：这门课虽然在教 Servlet 等底层技术，但所有的代码规范、MVC 思想以及分层架构，全都是在为你未来无缝衔接 Spring Boot 企业级框架，以及迈向更前沿的 AI 智能体与大模型应用开发 打下最坚实的地基。

    🌱 寄语¶

    计算机科学是一门真正的“手艺活”。看懂了不等于会写，会写了不等于能排错，会用 AI 生成代码也不等于你能驾驭系统的整体架构。

    在这里，我不希望你死记硬背任何一段代码。我希望你通过这份文档，理解代码背后的流转逻辑，学会看懂控制台的每一行报错（Exception），享受每一次把 Bug 揪出来（Debug）的成就感，并学会如何优雅地向 AI 工具提问。

    这份教程是“活”的，它会随着技术的发展和大家的反馈不断迭代。如果在学习过程中发现了错漏，或者有更好的实战想法，随时欢迎提出。
