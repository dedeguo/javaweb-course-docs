## 📚 分阶段学习指南

### **第一阶段（1-2周）：环境准备与效能革命**

**目标**：不纠结手动配置，建立现代化开发工作流

**理论学习（2学时）**：
- C/S vs B/S架构本质区别
- Maven依赖管理核心（pom.xml结构）
- **重点**：通义灵码/Cursor插件安装与提示词工程基础

**实验任务（2学时）**：
1. 基于教师提供的集成包5分钟配置完JDK+Idea+Tomcat
2. 安装AI编程助手插件
3. 用AI生成Hello Servlet代码并解析HTTP请求结构
4. **交付物**：AI生成代码截图 + HTTP包结构分析报告

**学习建议**：此阶段不要钻配置细节，信任Spring Boot的"约定大于配置"，把精力花在熟悉AI工具上

---

### **第二阶段（3-4周）：Web底层原理速成**

**目标**：理解"请求-响应"模型，解决登录状态存储问题

**理论学习（6学时）**：
- Servlet生命周期（了解即可）
- **HTTP协议深度解析**：Request/Response、状态码、Header
- **核心重点**：Cookie与Session机制（为后续鉴权打基础）
- JSON序列化与Jackson工具库

**实验任务（2学时）**：
1. 实现用户登录功能，用HttpSession存储用户信息
2. 使用Postman测试JSON接口
3. **交付物**：登录接口源码 + Session验证逻辑 + Postman测试截图

**删减说明**：**不讲JSP**、**不讲Servlet标签库**、**不讲纯Servlet CRUD** —— 这些已过时

---

### **第三阶段（5-7周）：Spring Boot核心攻坚**

**目标**：跳过SSM地狱配置，直接上手企业级开发

**理论学习（8学时）**：
- Spring Boot自动配置原理与启动流程
- **IoC与DI实战**：@Autowired、@Service、@Component
- Spring MVC核心注解：@RestController、@RequestMapping、@RequestBody、@PathVariable
- **RESTful API设计规范**：资源命名、HTTP方法语义、统一结果封装Result&lt;T&gt;
- 前后端联调：Postman/Apifox高级用法

**实验任务（4学时）**：
1. 初始化Spring Boot项目（使用Spring Initializr）
2. 设计航班查询、预订接口
3. 实现Controller→Service→Mapper分层逻辑（先Mock数据）
4. **交付物**：可返回JSON的航班API接口集合

**AI赋能点**：用AI生成Controller模板、Service层单元测试代码

---

### **第四阶段（8-9周）：国产数据库实战**

**目标**：掌握Java对象与openGauss的交互能力

**理论学习（6学时）**：
- ORM思想与MyBatis-Spring-Boot-Starter整合
- **信创重点**：openGauss数据库特性与JDBC配置
- Mapper接口与XML/注解双模式映射
- Service层事务控制：@Transactional实战
- **业务场景**：航班余票扣减的并发控制

**实验任务（2学时）**：
1. 配置DataSource连接远程openGauss（教师提供账号）
2. 编写FlightMapper实现真实CRUD
3. 将数据库层集成到上周的API中
4. **交付物**：数据库连接成功截图 + 基于真实数据的航班查询接口

**注意事项**：openGauss兼容性可能与MySQL有细微差别，遇到问题优先问AI或查阅官方文档

---

### **第五阶段（10-11周）：AI大模型集成**

**目标**：让应用"长脑子"，掌握第三方API集成能力

**理论学习（4学时）**：
- Java HTTP客户端：RestTemplate vs WebClient（响应式）
- **大模型接入**：DeepSeek API鉴权流程（API Key管理）
- **Prompt工程**：在代码中构建高效提示词的技巧
- 处理AI响应：Stream流式响应 vs 全量响应
- Markdown解析与前端展示方案

**实验任务（2学时）**：
1. 申请DeepSeek API Key（教师统一申请或指导）
2. 编写AIClient工具类封装API调用
3. 实现"输入目的地 → 返回AI生成的旅行攻略"功能
4. **交付物**：AI调用工具类源码 + API响应处理逻辑

**挑战点**：JSON解析、API超时处理、Token成本控制

---

### **第六阶段（12-14周）：综合项目整合**

**目标**：串联所有知识点，完成可演示的系统

**理论学习（6学时）**：
- 系统设计回顾：E-R图、API文档编写（Swagger）
- 模块开发：
  - 用户模块：注册登录 + 拦截器鉴权
  - 航班模块：条件查询 + 机票预订 + 余票管理
  - **特色模块**：AI旅行助手深度集成
- **部署上线**：Maven打包Jar包 + 龙蜥OS运行指令

**实验任务（4学时）**：
1. 整合所有模块，完成端到端联调
2. 实现前端页面（教师提供模板，学生专注后端）
3. 在龙蜥OS上部署演示
4. **交付物**：完整的智能航班管理系统 + 演示视频

## 📊 考核方式详解

### **总成绩构成（100%）**

| 考核模块 | 占比 | 考核形式 | 评分要点 |
| :--- | :--- | :--- | :--- |
| **平时成绩** | 50% | 过程性评价 | 课堂互动 + 实验代码 + AI应用报告 |
| **期末项目** | 50% | 终结性评价 | 智能航班系统功能完整度 |

---

### **平时成绩（50分）**

#### 1. 课堂参与与考勤（10分）
- **评分方式**：教师课堂记录
- **加分项**：主动提问、分享AI使用技巧、帮助同学解决问题
- **减分项**：缺勤、迟到、课堂刷手机

#### 2. 实验代码提交（20分）
- **评分方式**：Git仓库Commit记录分析
- **重点考察**：
  - **Commit频率**：每周至少2-3次提交，反对期末突击
  - **代码规范性**：命名、注释、分层合理性
  - **功能完整性**：是否按实验要求实现
- **防抄袭机制**：代码相似度检测 + 抽查答辩

#### 3. AI应用报告（20分）
- **报告标题**：《AI API集成心得与实践》
- **强制内容**：
  - Prompt调试过程记录（至少3个版本迭代）
  - JSON解析遇到的坑点及解决方案
  - AI工具使用前后开发效率对比（量化数据）
  - 对AI生成代码的质量评估与人工优化点
- **评分标准**：
  - 真实性：是否有真实调试记录（10分）
  - 深度：问题分析是否透彻（5分）
  - 创新性：是否有独特见解（5分）

---

### **期末项目考核（50分）**

#### **项目要求：智能航班管理系统**

**基础功能（60分 = 30分）**：
- 用户模块：注册、登录、Session鉴权（10分）
- 航班模块：增删改查、条件搜索（10分）
- 预订模块：机票预订、余票扣减、事务控制（10分）

**信创适配（+20分 = 10分）**：
- 成功连接openGauss数据库（5分）
- 在龙蜥OS环境部署运行（5分）

**AI功能（+20分 = 30分）**：
- **基础集成**：成功调用DeepSeek API（10分）
- **业务融合**：AI旅行助手与航班业务逻辑自然结合（10分）
- **体验优化**：流式响应、Markdown渲染、错误处理（10分）

#### **答辩要求**
- 演示时间：10分钟演示 + 5分钟问答
- 必须展示：Git提交历史、AI工具使用场景、openGauss连接证明
- **加分项**：超出大纲的创新功能（如支付集成、短信通知等）

## 🎯 学习建议与避坑指南

### **✅ 应该做的**
1. **拥抱AI，而非依赖AI**：用AI生成样板代码，但必须亲手修改和调试
2. ** early & often**：从第一周就频繁提交Git，养成良好习惯
3. **先跑通，再优化**：遇到问题时，先让代码work，再追求优雅
4. **多读官方文档**：Spring Boot和MyBatis的中文文档非常完善
5. **组队学习**：实验环节鼓励2-3人小组，但AI报告必须独立完成

### **❌ 应该避免的**
1. **不要钻牛角尖**：配置问题30分钟搞不定就问老师或AI
2. **不要抄袭代码**：Git Commit记录会暴露一切，相似度过高直接0分
3. **不要期末突击**：平时不学，最后两周无法完成AI集成
4. **不要忽视基础**：HTTP原理和Session机制是前后端交互的根本
5. **不要只看不练**：不亲手敲代码，永远学不会编程
