# 学习路线与考核说明

本页用于把《教学大纲》落到“每周学什么、做什么、交什么”的可执行安排上；如与课堂实际安排有差异，以教师当周通知为准。

---

## 一、课程路线（6 个阶段）

!!! abstract "课程主线：AI 赋能的航班管理系统"
    全课程围绕同一个项目逐步迭代：先理解 Web 底层，再用 Spring Boot + MyBatis 快速实现业务，最后完成 AI 集成与部署演示。

1. **第1章（2 学时）**：环境与工具就绪（Maven / Git / AI 助手）
2. **第2章（6 学时）**：Web 底层原理（HTTP、Request/Response、Cookie/Session、JSON）
3. **第3章（8 学时）**：Spring Boot 核心开发（IoC/DI、Controller、RESTful API、接口测试）
4. **第4章（4 学时）**：数据持久化与信创（MyBatis、事务、openGauss）
5. **第5章（6 学时）**：AI 集成与智能体基础（HTTP Client、LLM API、Tool Calling/MCP）
6. **第6章（6 学时）**：综合项目串联（功能闭环、部署与演示）

## 三、考核方式（以教学大纲为准）

!!! info "成绩构成（100%）"
    - **平时成绩（40%）**：课堂表现/考勤（30%） + 作业与实验（Git 提交记录）（40%） + AI 应用报告（30%）
    - **期末考试（60%）**：机考（闭卷），含编程题

---

## 四、如何使用这套电子教材（推荐节奏）

1. **课前 10 分钟**：先看每章的“导读页”，明确“本章三件事”和“不讲什么”。
2. **课中**：跟着“示例/实验步骤”走，遇到报错先学会定位（HTTP、日志、断点）。
3. **课后**：把代码推送到 Git，并在实验报告里补齐截图与说明（做到“可复现”）。

---

## 五、提交物与命名建议（减少返工）

!!! tip "建议统一规范（可直接作为课堂要求）"
    - 每次实验至少 3 次提交：`feat`（实现）/ `fix`（修复）/ `docs`（补文档或截图说明）
    - 截图命名：`lab2-login-ok.png`、`lab2-cookie.png`（能看懂即可）
