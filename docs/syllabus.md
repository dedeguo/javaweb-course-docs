**二、课程目标**

通过本课程的学习，使学生达到以下目标：

* **课程目标1（核心原理）：** 理解B/S架构工作原理，掌握HTTP协议、Session会话机制；理解Spring容器（IoC/DI）与AOP的基本思想；  
* **课程目标2（工程能力与效能）：** 熟练掌握**Spring Boot \+ MyBatis** 框架进行RESTful API开发；掌握Maven依赖管理；**熟练使用AI辅助编程工具（如通义灵码/Cursor）生成样板代码、编写单元测试及辅助Debug，提升开发效能**；  
* **课程目标3（信创与AI集成）：** 能够连接国产数据库（openGauss）进行开发；掌握**HTTP Client调用AI大模型（如DeepSeek）** 并处理JSON数据的能力；  
* **课程目标4（综合素质）：** 具备良好的代码规范意识，体验国产软硬件生态，培养解决复杂工程问题的创新思维。

**四、教学内容及要求**

**（一）理论教学（32学时）**

**第1章 Web开发基础与效能工具（2学时）**

* **目标**：不纠结手动配置，建立Maven工程概念，**学会用AI工具“偷懒”**。  
* **内容**：  
  1. C/S与B/S架构区别。  
  2. Maven依赖管理核心概念（pom.xml）。  
  3. **开发环境秒级就绪**：基于教师提供的集成包/镜像配置JDK、Idea、Tomcat。  
  4. **开发效能提升**：  
     * IDE必备插件：Lombok（简化代码）、MyBatisX（跳转映射）。  
     * **AI辅助编程**：安装**通义灵码/Cursor**，演示如何用AI生成Getter/Setter、快速生成单元测试代码以及解释复杂代码。  
* *教学策略*：演示为主，现场对比“手写代码”与“AI生成代码”的效率差异。

**第2章 Web底层原理：Servlet与HTTP（6学时）**

* **目标**：理解“请求-响应”模型，解决“登录状态怎么存”的问题。  
* **内容**：  
  1. Servlet生命周期（仅做了解）。  
  2. **核心重点**：HTTP协议，HttpServletRequest与HttpServletResponse的使用。  
  3. **难点突破**：Cookie与Session机制（为后续Spring Security/拦截器打基础）。  
  4. JSON数据格式与序列化（Jackson）。  
* *删减说明*：**不讲**JSP、**不讲**Servlet的高级标签、**不讲**纯Servlet的CRUD。

**第3章 Spring Boot核心与RESTful API（8学时）**

* **目标**：跳过SSM繁琐配置，直接上手企业级开发。  
* **内容**：  
  1. Spring Boot“约定大于配置”原理。  
  2. **IoC与DI**：通过@Autowired和@Service理解依赖注入。  
  3. Spring MVC核心注解：@RestController, @RequestMapping, @RequestBody。  
  4. **接口设计规范**：RESTful风格API设计与统一结果封装（Result\<T\>）。  
  5. 简单的前后端联调（使用Postman/Apifox测试接口）。

**第4章 数据持久化：MyBatis与信创数据库（6学时）**

* **目标**：掌握Java对象与国产数据库的交互。  
* **内容**：  
  1. ORM思想与MyBatis-Spring-Boot-Starter整合。  
  2. **信创实战**：连接远程**openGauss**数据库（教师提前建好库）。  
  3. Mapper接口与XML/注解映射。  
  4. Service层的事务控制（@Transactional）。  
  5. **实战场景**：航班数据的增删改查与余票扣减逻辑。

**第5章 AI集成与外部接口调用（4学时）**

* **目标**：让应用“长脑子”，掌握第三方API集成能力。  
* **内容**：  
  1. Java中的HTTP客户端工具（RestTemplate 或 WebClient）。  
  2. **大模型接入**：DeepSeek/OpenAI API 鉴权与调用流程。  
  3. **Prompt工程**：如何在代码中构建高效的Prompt（提示词）。  
  4. 处理AI流式响应（Stream）或全量响应。  
  5. Markdown格式数据的解析与前端展示方案。

**第6章 综合项目：智能航班管理系统（6学时）**

* **目标**：串联所有知识点，完成一个可演示的系统。  
* **内容**：  
  1. **系统设计**：E-R图回顾，API接口文档编写。  
  2. **模块开发**：  
     * 用户模块（登录/注册 \+ 拦截器鉴权）。  
     * 航班模块（条件查询、机票预订）。  
     * **特色模块**：AI旅行助手（根据目的地生成游玩攻略）。  
  3. **部署上线**：Maven打包（Jar包）与在龙蜥OS上的运行指令。

**（二）实验教学（16学时）**

**实验策略**：所有实验围绕同一个“航班系统”逐步迭代，而不是做零散的练习。

| 序号 | 实验名称 | 学时 | 内容提要 | 成果物 |
| :---- | :---- | :---- | :---- | :---- |
| **1** | **Web环境与AI助手配置** | 2 | 配置Maven/Idea；**安装并配置AI编程插件（如通义灵码）；尝试使用AI生成Hello Servlet代码**；分析HTTP包结构。 | AI生成代码截图、运行截图 |
| **2** | **Session登录与接口测试** | 2 | 实现用户登录功能；使用Session存储用户信息；Postman测试JSON接口。 | 登录接口代码、Session验证 |
| **3** | **Spring Boot 航班API开发** | 4 | 初始化Spring Boot项目；设计航班查询、预订的RESTful接口；实现Controller与Service层逻辑。 | 能够返回Mock数据的API接口 |
| **4** | **MyBatis对接openGauss** | 2 | 配置DataSource连接openGauss；编写Mapper实现航班表的真实CRUD；集成到上一出的API中。 | 数据库连接成功、真实数据接口 |
| **5** | **AI智能助手集成实战** | 6 | **(重点)** 申请DeepSeek API Key；编写工具类调用API；实现“输入目的地 \-\> 返回AI生成的旅行计划”功能；项目整合与演示。 | **智能航班系统Demo** |

**五、考核方式（改革版）**

**总成绩 (100%) \= 平时成绩 (50%) \+ 期末项目考核 (50%)**

1. **平时成绩 (50%)**：  
   * **课堂/考勤 (10%)**：鼓励提问与互动。  
   * **实验代码 (20%)**：每次实验Git提交记录（重点看Commit频率，防止抄袭）。  
   * **AI应用报告 (20%)**：不考死记硬背。要求学生提交一份《AI API集成心得》，包含Prompt调试过程、遇到的JSON解析坑点及解决方案。  
2. **期末项目考核 (50%)**：  
   * **形式**：大作业答辩或机考。  
   * **要求**：提交“智能航班管理系统”。  
   * **评分标准**：  
     * 基础功能（登录、CRUD）：60分。  
     * 信创环境（连接openGauss）：+10分。  
     * **AI功能（调用大模型成功）：+30分。**

**六、教学资源与保障**

1. **教材**：建议使用自编讲义或《Spring Boot实战》类技术书籍，传统Java Web教材已不适用。  
2. **软硬件支持**：  
   * 服务器：教师需提前在私有云/服务器搭建好openGauss数据库，并开通学生账号。  
   * 前端资源：教师提供Vue或HTML+jQuery的静态页面模板，**让学生专注于后端逻辑**。  
   * API Key：建议教师申请统一的API Key（或指导学生申请）用于教学。
